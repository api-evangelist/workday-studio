<p><em>by Jim Hu, Workday San Francisco</em></p>

<p>Fragment transitions with RecyclerView items “sliding apart” or “Split”</p>

<div style="margin: auto ; width: 50%;">
<img height="360" src="/assets/android/transition.gif" title="Transition" width="640" />
</div>
<p><br />
We wanted to animate an inbox-like feature’s transition from <strong>List</strong> to
<strong>Details</strong> with  Material transitions that follows the Material Design
guidelines stated
<a href="https://material.io/guidelines/motion/material-motion.html#material-motion-implications-of-motion">here</a>.</p>

<!--more-->

<p>The example provided by the guideline is Google’s Android Inbox App, it has the
following characteristics:</p>

<ul>
  <li>The selected inbox item and all above it slide out from the <em>top</em>.</li>
  <li>All items under the selected slides out from the <em>bottom</em>.</li>
  <li>Detail page slides up uniformly from the <em>bottom</em></li>
  <li>Loading should start immediately after click <em>and</em> as soon as transition starts</li>
</ul>

<p><strong>We will focus on the “Sliding” for now</strong></p>

<p>Note: Some examples are in Kotlin</p>

<hr />

<h2 id="transition-terms">Transition terms</h2>

<p>The transition terms are confusing at first glance, using our example, here’s
the 4 transition in layman’s terms, in their actual <strong>triggered order</strong>:</p>

<h3 id="opening-an-item-from-the-list">Opening an item from the List:</h3>

<ul>
  <li>ExitTransition — <strong>List’s</strong> transition for <strong>List→Detail</strong></li>
  <li>EnterTransition — <strong>Detail’s</strong> transition for <strong>List → Detail</strong></li>
</ul>

<h3 id="clicking-back-or-returning-from-detail-to-list">Clicking back or returning from Detail to List:</h3>

<ul>
  <li>ReturnTransition — <strong>Detail’s</strong> transition for <strong>Detail → List</strong></li>
  <li>ReenterTransition — <strong>List’s</strong> transition for <strong>Detail → List</strong></li>
</ul>

<p>*Reverse animations are also autogenerated for the target’s <strong>reverse</strong> action
(can be overridden by setting them yourselves), i.e.</p>

<ul>
  <li>Setting List’s ExitTransition will autogenerate it’s ReenterTransition</li>
  <li>Setting Detail’s EnterTransition will autogenerate it’s ReturnTransition</li>
</ul>

<h3 id="in-code-itll-look-something-like-this">In code it’ll look something like this</h3>
<div class="language-kotlin highlighter-rouge"><div class="highlight"><pre class="highlight"><code><span class="n">detailFragment</span><span class="p">.</span><span class="n">enterTransition</span> <span class="p">=</span> <span class="n">Slide</span><span class="p">()</span>
<span class="c1">// If you don't want the Fade on the way back 
</span><span class="n">detailFragment</span><span class="p">.</span><span class="n">returnTransition</span> <span class="p">=</span> <span class="k">null</span> 
<span class="n">listFragment</span><span class="p">.</span><span class="n">exitTransition</span> <span class="p">=</span> <span class="n">Explode</span><span class="p">()</span>
<span class="c1">//listFragment.reenterTransition is auto generated reversed
</span></code></pre></div></div>

<p>There’s a few subtle but important prerequisites that must be set on the

<h3 id="breaker-of-chains--transition-groups">Breaker of chains — Transition Groups</h3>

<p>By default all views under a parent/ancestor  with a <strong>background</strong> set (even
transparent ones) will be <strong>automatically deemed a group</strong>. If you need to break
them up like we here with a RecyclerView as the shared-root-white-backgrounded
layout with transparent child Item views. You’ll need to set the layout with the
background to <code class="highlighter-rouge">transitionGroup=false</code>.</p>

<p>But on the other hand, since the Items are “background-less” themselves, to
prevent an out-of-body experience you’ll need to do the <strong>opposite</strong> and set <code class="highlighter-rouge">transitionGroup=true</code> on
the Item layouts for all the child views in that Item to move together.</p>

<h3 id="set-the-children-free--remove-parent-layout-boundaries">Set the children free! — Remove parent layout boundaries</h3>

<p>Items inside RecyclerViews are limited to animating inside their parent’s
boundaries, to let the items fly over the AppBar on top, we’ll need to set <code class="highlighter-rouge">clipChildren=”false”</code> on
<strong>all parents/ancestors</strong> up the view hierarchy.</p>

<hr />

<h2 id="crafting-the-transition--the-vertical-split">Crafting the Transition — The vertical “Split”</h2>

<p>Since Android 5.0 there’s 3 available default visibility transitions available:
<a href="https://developer.android.com/reference/android/transition/Explode.html">Explode</a>, <a href="https://developer.android.com/reference/android/transition/Fade.html">Fade</a>, <a href="https://developer.android.com/reference/android/transition/Slide.html">Slide</a>, but wait, no <strong>Split</strong>?!? 😱</p>

<p>Luckily, if you look closely (in one dimensional vision 👀), you’ll realize a
Split is actually just a one dimensional Explode! Setting the Explode
transition’s epic-center to an horizontal line right under the selected item
will let the view explode vertically.</p>
<div class="language-kotlin highlighter-rouge"><div class="highlight"><pre class="highlight"><code><span class="k">private</span> <span class="k">fun</span> <span class="nf">getListFragmentExitTransition</span><span class="p">(</span><span class="n">itemView</span><span class="p">:</span> <span class="n">View</span><span class="p">):</span> <span class="n">Transition</span> <span class="p">{</span>
      <span class="kd">val</span> <span class="py">epicCenterRect</span> <span class="p">=</span> <span class="n">Rect</span><span class="p">()</span>
      <span class="c1">//itemView is the full-width inbox item's view
</span>      <span class="n">itemView</span><span class="p">.</span><span class="n">getGlobalVisibleRect</span><span class="p">(</span><span class="n">epicCenterRect</span><span class="p">)</span>
      <span class="c1">// Set Epic center to a imaginary horizontal full width line under the clicked item, so the explosion happens vertically away from it
</span>      <span class="n">epicCenterRect</span><span class="p">.</span><span class="n">top</span> <span class="p">=</span> <span class="n">epicCenterRect</span><span class="p">.</span><span class="n">bottom</span>
      <span class="kd">val</span> <span class="py">exitTransition</span> <span class="p">=</span> <span class="n">Explode</span><span class="p">()</span>
      <span class="n">exitTransition</span><span class="p">.</span><span class="n">epicenterCallback</span> <span class="p">=</span> <span class="kd">object</span> <span class="err">: </span><span class="nc">Transition</span><span class="p">.</span><span class="n">EpicenterCallback</span><span class="p">()</span> <span class="p">{</span>
          <span class="k">override</span> <span class="k">fun</span> <span class="nf">onGetEpicenter</span><span class="p">(</span><span class="n">transition</span><span class="p">:</span> <span class="n">Transition</span><span class="p">):</span> <span class="n">Rect</span> <span class="p">{</span>
              <span class="k">return</span> <span class="n">epicCenterRect</span>
          <span class="p">}</span>
      <span class="p">}</span>
<span class="p">}</span>
</code></pre></div></div>
<p>and Voilà!</p>

<p>But the items are not <em>sliding</em> away, but still *exploding *i.e. Items not
flying together. Nobody’s inbox should explode (at least not visually..)</p>

<h3 id="propagation">Propagation</h3>

<p>The <em>explode</em> effect is actually achieved by <strong>propagation</strong>, which delays view
animation according to the view’s distance from the epic-center, i.e. start
later if the item’s closer to the epic-center. As we want to just <em>Slide</em>, we
disable Propagation by: <code class="highlighter-rouge">exitTransition.setPropagation(null)</code></p>

<p>And~~ now we have the <strong>Slide</strong>! So smooth~</p>

<h2 id="additional-stuff--adding-fadeout">Additional stuff — Adding FadeOut</h2>

<p>You may have probably noticed that we also incorporated “Fade Out” into the
transition. Ideally one could just utilize
<a href="https://developer.android.com/reference/android/transition/TransitionSet.html">TransitionSet</a>s
and stuff them together to your heart’s content. But due to a
<a href="https://issuetracker.google.com/issues/37118689">bug</a>, Fragment Exit/Reenter
transitions to not work with TransitionSets (Enter/Return works). Hence for
whoever that has this problem, we could just inherit the Transition and
manipulate the underlying AnimationSet, the following is the full inherited
class:</p>

<p>Note: <strong>onAppear</strong> is for Enter/Reenter, <strong>onDisappear</strong> is for Exit/Return</p>

    <span class="n">init</span> <span class="p">{</span>
